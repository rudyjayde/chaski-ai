En la Figura 1 se muestra la organización del frontend dentro del directorio raíz del proyecto. El frontend fue desarrollado con HTML5, CSS3 y JavaScript Vanilla, sin dependencia de frameworks, lo que permite mayor control sobre el rendimiento y la carga inicial. Se adoptó una arquitectura modular por funcionalidad: cada módulo del panel administrador (manifests, drivers, vehicles, queue, trips, gps, reports, assistant) reside en su propia subcarpeta dentro de admin/, agrupando su HTML, hoja de estilos y lógica de negocio. Los estilos compartidos se centralizan en admin/index.css y css/main.css, mientras que la navegación utiliza rutas canónicas absolutas (/admin/manifests, /admin/drivers, etc.) servidas directamente por el servidor Express, eliminando las extensiones .html de las URLs.

En la Figura 2 se muestra la estructura del backend implementado con Node.js y Express. Los módulos se organizan dentro del directorio backend/ siguiendo el patrón MVC. La carpeta routes/ contiene un archivo por cada módulo de negocio: auth, drivers, vehicles, queue, manifests, trips, reports, assistant, gps y gps-devices, cada uno exponiendo sus endpoints REST. La carpeta middleware/ centraliza la verificación del token JWT y el control de acceso por roles (administrador y conductor). La configuración de la base de datos PostgreSQL se gestiona mediante un pool de conexiones en config/db.js. El archivo server.js incorpora un middleware de URLs limpias que resuelve rutas como /admin/manifests hacia admin/manifests/index.html sin requerir la extensión. Adicionalmente, gps-server.js levanta un servidor TCP en el puerto 2223 para recibir tramas en protocolo Codec 8 del dispositivo GPS Teltonika FMC130.

En la Figura 3 se muestra el despliegue de la aplicación en Render, plataforma de nube que proporciona infraestructura como servicio (PaaS). El backend Node.js se despliega como un Web Service conectado al repositorio GitHub rudyjayde/chaski-ai, configurado con Auto-Deploy: cada vez que se realiza un push a la rama main, Render detecta el cambio automáticamente e inicia un nuevo despliegue sin intervención manual. Las variables de entorno sensibles (credenciales de PostgreSQL, JWT secret, API key de Anthropic) se gestionan desde el panel de Environment de Render, sin exponerlas en el código fuente. La URL pública del sistema es https://chaski-ai.onrender.com, accesible desde cualquier dispositivo con conexión a internet, lo que facilita las demostraciones y evaluaciones del sistema sin necesidad de configuración local.
