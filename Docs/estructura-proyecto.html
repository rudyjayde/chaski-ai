En la Figura 1 se muestra la organización del frontend dentro del directorio raíz del proyecto. Para este sistema, se separaron las vistas según el rol del usuario en carpetas independientes. La carpeta admin/ concentra los nueve módulos del administrador: dashboard con KPIs en tiempo real, gestión de conductores, manifiestos digitales, cola FIFO de salida, historial de viajes, flota de vehículos, reportes e insights, y el asistente IA; la carpeta driver/ agrupa los cuatro módulos del conductor. La carpeta css/ contiene el sistema de diseño global con variables Glassmorphism, y la carpeta img/ los recursos gráficos. Esta arquitectura de módulos independientes permite mantener la separación de responsabilidades y facilita la escalabilidad del frontend sin dependencia de frameworks adicionales.

En la Figura 2 se muestra la estructura del backend implementado con Node.js y Express.js. Los módulos se organizaron dentro del directorio backend/, separando cada responsabilidad en capas. El archivo server.js contiene el punto de entrada, la configuración de CORS, Socket.io y el servidor TCP para el dispositivo GPS; config/db.js administra el pool de conexiones a PostgreSQL; la carpeta middleware/ contiene la verificación de tokens JWT y el control de roles; y la carpeta routes/ agrupa los nueve módulos de la API REST: autenticación, conductores, vehículos, cola de salida, manifiestos, viajes, reportes y el asistente de inteligencia artificial. Esta separación permite mantener una arquitectura limpia, modular y fácilmente escalable.

En la Figura 3 se muestra la arquitectura de despliegue de Chaski AI 2.0. El sistema se aloja en un servidor VPS de DigitalOcean con Ubuntu 22.04, donde Nginx actúa como proxy inverso gestionando el tráfico HTTPS hacia el servidor Node.js/Express en el puerto 3005. La base de datos PostgreSQL 15 corre en el mismo servidor. Los dispositivos GPS Teltonika FMC130, instalados en cada vehículo de la asociación, envían paquetes de telemetría mediante protocolo Codec 8 sobre TCP al puerto 2223; el servidor los decodifica y persiste en la tabla gps_positions, retransmitiendo las posiciones al dashboard del administrador en tiempo real mediante Socket.io. El módulo de inteligencia artificial se integra con la API de Anthropic Claude para procesar consultas en lenguaje natural sobre el estado operativo de la flota.
