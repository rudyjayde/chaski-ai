  1. RESUMEN DEL SISTEMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  CHASKI AI es una plataforma web para digitalizar y optimizar la gestión
  operativa de la Asociación de Transportistas Virgen de Fátima (ATIPCAR),
  que opera la ruta Juli–Puno (98.5 km) con más de 60 vehículos socios.

  PROBLEMA QUE RESUELVE:
    · Control de cola de salida hecho a mano (conflictos entre conductores)
    · Manifiestos de viaje en papel (se pierden, ilegibles, lentos)
    · Sin visibilidad GPS de la flota en tiempo real
    · Sin reportes de ingresos ni desempeño de conductores

  TECNOLOGÍAS USADAS:
    · Frontend  : HTML5 + CSS3 + JavaScript Vanilla
    · Backend   : Node.js + Express.js (puerto 3005)
    · Base datos: PostgreSQL (15 tablas)
    · Mapas     : Leaflet.js
    · Gráficos  : Chart.js
    · Animación : Three.js (landing page), GSAP
    · Auth      : JWT + bcryptjs
    · IoT       : Teltonika FMC130 (protocolo Codec 8, servidor TCP puerto 2223)
    · IA        : Claude API de Anthropic — claude-haiku-4-5 (IMPLEMENTADO)
    · Deploy    : Render (PaaS) — https://chaski-ai.onrender.com

  RUTA DE LA ASOCIACIÓN: Juli (Chucuito, Puno) → Puno · 98.5 km · S/ 7.00

  4. BACKEND — API REST implementada
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ESTADO: Completamente implementado y desplegado en Render.
  Puerto: 3005 (local) / asignado por Render (producción)
  URL producción: https://chaski-ai.onrender.com

  DEPENDENCIAS INSTALADAS (package.json):
    · express           → Servidor HTTP y API REST
    · pg                → Conexión a PostgreSQL
    · jsonwebtoken      → Autenticación JWT
    · bcryptjs          → Hash de contraseñas
    · uuid              → IDs únicos
    · dotenv            → Variables de entorno
    · cors              → CORS para el frontend
    · @anthropic-ai/sdk → Claude API (Anthropic) — asistente IA ✅

  CARACTERÍSTICAS DEL SERVIDOR (server.js):
    · Sirve archivos estáticos del frontend desde la raíz del proyecto
    · Middleware de URLs limpias: /admin/manifests → admin/manifests/index.html
    · Ruta canónica: /admin y /admin/ redirigen a /admin/dashboard
    · Health check en GET /api/health (verifica conexión a BD)

  ENDPOINTS IMPLEMENTADOS:

  POST /api/auth/login         → Valida credenciales, retorna JWT
  GET  /api/auth/me            → Datos del usuario autenticado

  GET    /api/drivers          → Lista conductores
  GET    /api/drivers/:id      → Detalle de conductor
  POST   /api/drivers          → Crear conductor
  PUT    /api/drivers/:id      → Editar conductor

  GET    /api/vehicles         → Lista vehículos
  GET    /api/vehicles/:id     → Detalle de vehículo
  POST   /api/vehicles         → Crear vehículo
  PUT    /api/vehicles/:id     → Editar vehículo

  GET    /api/queue            → Cola del día
  POST   /api/queue            → Registrar en cola
  PUT    /api/queue/:id/status → Cambiar estado
  GET    /api/queue/position   → Posición del conductor autenticado

  GET    /api/manifests        → Lista manifiestos (filtros: fecha, estado)
  POST   /api/manifests        → Crear manifiesto
  PUT    /api/manifests/:id    → Actualizar / cerrar manifiesto
  POST   /api/manifests/:id/passengers → Agregar pasajero

  GET    /api/trips            → Historial de viajes
  POST   /api/trips            → Iniciar viaje
  PUT    /api/trips/:id        → Finalizar viaje

  GET    /api/reports/...      → KPIs, ranking, alertas de velocidad

  POST   /api/assistant        → Pregunta al asistente Claude IA

  GET    /api/gps              → Posiciones GPS en tiempo real
  GET/POST /api/gps-devices    → Dispositivos Teltonika registrados

  GET/POST /api/communications → Comunicados a conductores
  DELETE /api/communications/:id → Eliminar comunicado

  GET    /api/health           → Estado del servidor y conexión a BD

  SERVIDOR TCP (gps-server.js):
    · Puerto 2223
    · Recibe tramas Codec 8 del Teltonika FMC130
    · Decodifica lat, lon, velocidad, heading, acelerómetro
    · Guarda en tabla gps_positions de PostgreSQL
    · Genera alertas si velocidad > 90 km/h

  5. BASE DE DATOS — PostgreSQL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ESTADO: Implementada y en producción en Render.

  TABLAS Y SU PROPÓSITO:

  associations        → Asociaciones (Virgen de Fátima / ATIPCAR)
  companies           → Empresas dentro de cada asociación (5 en demo)
  vehicle_types       → Tipos de vehículo (HiAce 12p, Sprinter 19p, Master 15p)
  users               → Usuarios del sistema (admin, driver) + JWT auth
  vehicles            → Flota con campo gps_device_id (FMC130)
  drivers             → Conductores socios vinculados a usuario y vehículo
  routes              → Rutas definidas (Juli→Puno, Puno→Juli · 98.5 km · S/7.00)
  exit_queue          → Cola de salida diaria FIFO con estados
  manifests           → Manifiestos de viaje (vehículo, conductor, ruta)
  manifest_passengers → Pasajeros de cada manifiesto (DNI, asiento, pago)
  trips               → Viajes realizados (velocidad media, máxima, duración)
  gps_positions       → Telemetría GPS del FMC130 (lat, lon, speed, altitud, sat)
  speed_alerts        → Alertas de velocidad (>90 km/h) con severidad y ACK
  route_alerts        → Alertas de desvío de ruta, paradas no programadas
  system_alerts       → Notificaciones del sistema para el panel admin

  JERARQUÍA DE DATOS:
    Asociación → Empresa → Vehículo → Driver → Trip → Manifest → Passengers
                                    ↑
                            gps_positions (FMC130)
                            speed_alerts
                            route_alerts

  6. IOT — INTEGRACIÓN TELTONIKA FMC130
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  DISPOSITIVO: Teltonika FMC130
    · Conectividad: 4G LTE Cat 1 / 3G / 2G (fallback automático)
    · GNSS: GPS + GLONASS + BeiDou + Galileo (precisión 2–5 metros)
    · Protocolo: Codec 8 sobre TCP (binario comprimido)
    · Alimentación: 10–30V DC (batería del vehículo)
    · Memoria: 128 MB store & forward (guarda datos sin cobertura)
    · Acelerómetro: triaxial (detecta frenazos, aceleración agresiva, impactos)

  SERVIDOR TCP: backend/gps-server.js — Puerto 2223
